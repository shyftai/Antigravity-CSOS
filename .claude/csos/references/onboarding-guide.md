# Customer Onboarding Guide -- CS:OS

Best practices for onboarding new customers. Reference this during kickoff and the first 90 days.

---

## Pre-Kickoff Checklist

Complete before the kickoff meeting:

| Item | Owner | Status |
|------|-------|--------|
| Sales handoff doc received (goals, pain points, champion, deal context) | Sales > CSM | [ ] |
| CRM record updated with CSM assignment | CS Ops | [ ] |
| Customer account provisioned | CSM / Ops | [ ] |
| Admin credentials sent to customer | CSM | [ ] |
| Success criteria drafted (from sales notes) | CSM | [ ] |
| Kickoff meeting scheduled (<48h from close) | CSM | [ ] |
| Kickoff agenda sent to customer | CSM | [ ] |
| Internal stakeholders briefed (support, product if needed) | CSM | [ ] |
| Previous vendor/tool migration plan (if applicable) | CSM | [ ] |
| Billing and contract terms confirmed | Finance | [ ] |

## Kickoff Meeting Agenda Template

**Duration:** 45-60 minutes
**Attendees:** CSM, customer champion, key stakeholders, exec sponsor (optional)

```
1. Introductions (5 min)
   - CS team introduction
   - Customer team introductions and roles
   - Overview of the CS partnership model

2. Goals alignment (15 min)
   - Confirm business objectives (from sales handoff)
   - Define measurable success criteria
   - Agree on timeline for each goal
   - Identify potential blockers

3. Stakeholder mapping (5 min)
   - Confirm champion and their role
   - Identify exec sponsor
   - Identify day-to-day users
   - Agree on communication preferences

4. Onboarding plan walkthrough (10 min)
   - 30/60/90 day milestones
   - Training schedule
   - Key checkpoints and reviews
   - Go-live target date

5. Technical setup review (10 min)
   - Account access confirmation
   - Integration requirements
   - Data migration plan (if applicable)
   - Security/compliance requirements

6. Next steps and action items (5 min)
   - Confirm first training session date
   - Assign action items with owners
   - Set next check-in date
```

## Success Criteria Definition

Define before or during kickoff. Must be:

- **Specific:** "Reduce ticket resolution time by 30%" not "improve support"
- **Measurable:** tied to a metric you can track
- **Time-bound:** "within 60 days" not "eventually"
- **Customer-owned:** the customer agrees this matters to them
- **Achievable:** realistic given their setup and resources

Template:
```
## Success Criteria -- {Customer Name}

Primary outcome: {What the customer needs to achieve}
Metric: {How we measure it}
Baseline: {Current state}
Target: {Goal state}
Timeline: {By when}

Secondary outcomes:
1. {Outcome} -- measured by {metric} -- target: {value} by {date}
2. {Outcome} -- measured by {metric} -- target: {value} by {date}
```

| Role | Name | Title | Engagement level | Notes |
|------|------|-------|-----------------|-------|
| Champion | | | Primary contact, internal advocate | |
| Exec sponsor | | | Quarterly engagement, escalation path | |
| Day-to-day user 1 | | | Weekly engagement | |
| Day-to-day user 2 | | | Weekly engagement | |
| Technical lead | | | As needed (integrations, setup) | |
| Finance/procurement | | | Renewal conversations | |

**Multi-threading rules:**
- Never be single-threaded (1 contact only) -- always have 2+ engaged contacts
- If champion leaves, you should already know their replacement
- Exec sponsor should hear from you at least quarterly

## Time-to-Value Tracking

Track from contract signed to first value milestone:

| Event | Date | Days from start |
|-------|------|----------------|
| Contract signed | | Day 0 |
| Kickoff meeting | | Target: Day 2 |
| Account provisioned | | Target: Day 3 |
| First training | | Target: Day 7 |
| First login (by customer) | | Target: Day 3 |
| First workflow completed | | Target: Day 7 |
| First value milestone | | Target: Day 14 |
| Go-live declared | | Target: Day 14-21 |

**TTV target:** 14 days (configurable per workspace)

If TTV exceeds 21 days, investigate and document the blocker. Common delays:
- Customer resource availability
- Integration complexity
- Data migration issues
- Internal approval processes
- Technical prerequisites not met
