## Onboarding milestones

### Phase 1: Kickoff (Days 0-7)

| Milestone | Owner | Success criteria | Required |
|-----------|-------|-----------------|----------|
| Sales-to-CS handoff complete | AE + CSM | Notes, goals, stakeholders documented | Yes |
| Welcome email sent | CSM | Sent within 24h of close | Yes |
| Kickoff call scheduled | CSM | Within 3 business days of close | Yes |
| Kickoff call completed | CSM | Goals, timeline, stakeholders confirmed | Yes |
| Success plan draft created | CSM | Measurable outcomes defined | Yes |
| Technical requirements documented | CSM | Integration needs, data migration scope | Yes |

### Phase 2: Setup (Days 7-21)

| Milestone | Owner | Success criteria | Required |
|-----------|-------|-----------------|----------|
| Account provisioned | Support | All users have access | Yes |
| Integrations configured | Support + CSM | CRM, SSO, data sources connected | Yes |
| Data migration complete | Support | Historical data imported and verified | If applicable |
| Admin training delivered | CSM | Admin can manage users, settings | Yes |
| Core workflow training delivered | CSM | Team can perform primary use cases | Yes |

### Phase 3: Adoption (Days 21-60)

| Milestone | Owner | Success criteria | Required |
|-----------|-------|-----------------|----------|
| End-user training complete | CSM | 80%+ of licensed users trained | Yes |
| First value milestone achieved | CSM | Customer confirms value from product | Yes |
| Advanced feature training | CSM | Power users trained on advanced workflows | No |
| 30-day health check | CSM | Usage metrics reviewed, blockers cleared | Yes |
| Success plan updated | CSM | Progress documented, next goals set | Yes |

### Phase 4: Go-Live (Days 60-90)

| Milestone | Owner | Success criteria | Required |
|-----------|-------|-----------------|----------|
| Go-live confirmed | CSM | Customer self-identifies as "live" | Yes |
| Steady-state cadence set | CSM | Regular meeting schedule agreed | Yes |
| First QBR scheduled (Enterprise+) | CSM | Date confirmed, agenda drafted | Segment-dependent |
| Onboarding survey sent | CSM | NPS + onboarding satisfaction captured | Yes |
| Handoff to steady-state | CSM | Onboarding closed, regular CS begins | Yes |

## Go-live checklist

Before marking a customer as "onboarded":

- [ ] All integrations connected and syncing
- [ ] 80%+ of licensed users have logged in
- [ ] Primary use case is operational
- [ ] Customer has confirmed first value
- [ ] Success plan reviewed and accepted
- [ ] Steady-state meeting cadence agreed
- [ ] Support escalation path communicated
- [ ] Onboarding satisfaction score collected
